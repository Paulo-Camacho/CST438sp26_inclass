## Template for Pull Requests

- What (The Change):
    - Describe the "what" by summarizing the changed files, the functionality added, modified, or removed. Use clear, concise titles and include references to issue tickets.

- How (The Implementation):
    - Explain "how" the code achieves the goal, highlighting key architectural decisions, complex logic, or new dependencies.

- Why (The Rationale):
    - Detail "why" this change is necessary. This includes business justification, design choices, or links to technical requirements, providing context to reviewers.
